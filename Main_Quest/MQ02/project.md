# 🌬️ 풍력 발전기 스마트 예방 정비 시스템

## 1. 프로젝트 개요
* **목표:** 드론 기반 온디바이스 AI를 활용한 풍력 발전기 블레이드 손상(Damage) 및 오염(Dirt) 실시간 탐지.
* **배경:** 접근이 어려운 해상/산간 지역의 풍력 터빈 유지보수 효율성 극대화.

## 2. 모델 선정 및 앙상블 전략
5종의 베이스라인 모델 비교 후, 상호 보완적인 **하이브리드 앙상블 파이프라인**을 구축했습니다.

| 모델 | 역할 | 특징 |
| :--- | :--- | :--- |
| **YOLOv8s** | 돋보기 (Recall) | 전역 탐색 및 미세 결함 누락 방지 |
| **YOLOv12s** | 현미경 (Precision) | 정밀 타격 및 결함 분류 확신도 강화 |

* **시너지:** YOLOv8이 후보군을 추출하고, YOLOv12가 정밀 분석하는 2단계 프로세스를 통해 하드웨어 부하(GPU)는 낮추고 탐지 정확도는 높였습니다.

## 3. 기술적 고도화 과정 (Pipeline & Tuning)
* **데이터 신뢰도 개선:** 정상 이미지(Empty Label)를 1:1 비율로 주입하여 배경 오탐(False Alarm)을 기존 30%에서 14%로 감축.
* **실무적 안정성:** 고해상도 영상(1920x1080) 처리 시 발생한 OOM(Out of Memory) 문제를 `gc.collect()`와 `torch.cuda.empty_cache()`를 통한 메모리 관리로 해결.
* **학습 최적화:** Epoch를 20에서 30으로 확장하여 모델의 예측 확신도(Confidence Score)를 0.2~0.3 수준 향상.

## 4. 트러블 슈팅 및 인사이트
* **문제:** 외부 데이터셋(WTBD) 적용 시 정량 지표(mAP) 급락.
* **원인:** 학습 데이터와 외부 데이터 간의 도메인 갭(Domain Gap) 존재.
* **교훈:** 데모 영상의 시각적 결과물에 현혹되지 말고, 엄격한 데이터 도메인 기반의 정량적 검증이 선행되어야 함을 실증함.
